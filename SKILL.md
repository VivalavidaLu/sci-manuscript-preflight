# SCI Manuscript Preflight

## Purpose

Use this skill before submitting a biomedical, clinical, bioinformatics, immunology, oncology, single-cell, systematic review, or general SCI manuscript.

The goal is not to accuse the author of AI use. The goal is to catch submission-risk artifacts before peer review or editorial screening.

This skill focuses on:

1. AI residue and drafting artifacts.
2. Hallucinated, fabricated, or mismatched references.
3. Claim-to-citation support problems.
4. Figure, table, supplementary, and numbering consistency.
5. Figure, data, and statistical integrity risks.
6. Tabular source-data numerical integrity and submission release gating.
7. Methods, statistics, ethics, data availability, and reporting guideline gaps.
8. Target-journal readiness.

## Inputs

Accept any of the following:

- Manuscript `.docx`
- Manuscript `.pdf`
- Manuscript `.tex`
- Reference file `.bib`, `.ris`, `.enl`, `.xml`, `.nbib`, `.txt`
- Figure folder
- Source-data tables: `.xlsx`, `.xls`, `.xlsm`, `.xlsb`, `.csv`, `.tsv`
- Statistical scripts, notebooks, or analysis exports
- Supplementary files
- Cover letter
- Target journal instructions
- Reviewer checklist
- Response-to-reviewers draft

If the user provides only one file, perform the best possible partial preflight and explicitly state missing inputs.

## Operating Principles

- Treat the manuscript as confidential.
- Do not make definitive accusations of AI use.
- Do not treat AI detector scores as reliable evidence.
- Prefer verifiable checks: DOI, PMID, title, author, year, journal, figure number, table number, citation context.
- Distinguish verified errors from suspicious items requiring manual review.
- Do not silently rewrite scientific claims. Flag overclaims and propose conservative wording.
- For biomedical references, prioritize PubMed, Europe PMC, Crossref, DOI resolver, journal pages, arXiv, bioRxiv, medRxiv, OpenAlex, and Semantic Scholar.
- For clinical studies, check ethics, consent, trial registration, reporting guideline compliance, and data availability.
- For bioinformatics studies, check reproducibility, code availability, data accession, versioning, parameters, and benchmark design.
- For figure and data integrity, do not accuse authors of misconduct. Flag visible reuse, provenance gaps, statistical inconsistencies, and methods-results contradictions as submission risks requiring author verification.
- Treat deterministic table findings as signals requiring source-table verification, not as proof of error, fabrication, or intent.
- Never interpret an empty finding list as proof that the data are correct.
- Do not assign final `PASS` when expected source-data files failed to parse, central quantitative outputs lack traceable source data, or unresolved high-impact findings remain.
- Separate scanner output from scientific interpretation, preserve benign explanations, and require named author sign-off for unresolved manual items.

## Workflow

### Step 1: Inventory

List all provided files and classify them:

- Main manuscript
- Reference file
- Figures
- Tables
- Supplementary material
- Cover letter
- Journal instructions
- Other

Report missing but expected files.

### Step 2: Static Residue Scan

Run or emulate a scan for:

- Explicit AI residue in context: `as an AI language model`, `generated by ChatGPT`, `generated by Claude`, `generated by Gemini`, or `AI-generated response`.
- Placeholder text: `TODO`, `TBD`, `[insert citation]`, `[add reference]`, `[figure here]`, `请核实`, `待补充`, `占位`.
- Markdown remnants: raw `#`, `##`, bullet artifacts, code fences, pipe tables accidentally left in text.
- LaTeX remnants in Word/PDF text: `\cite{}`, `\ref{}`, `\label{}`, unresolved `??`.
- Editorial notes: `revise this`, `make this more scientific`, `polish`, `humanize`, `avoid AI detection`.
- Suspicious generic AI boilerplate: excessive `delve`, `underscore`, `crucial`, `multifaceted`, `plays a pivotal role`, `it is important to note`.

Classify findings as:

- Critical: explicit AI/prompt residue or author instruction accidentally retained.
- Major: placeholders, unresolved citations, visible Markdown/LaTeX artifacts.
- Minor: style risks or overly generic AI-like phrasing.

### Step 3: Reference Authenticity Audit

Extract all references and verify:

- DOI exists.
- PMID exists where expected.
- Title matches DOI/PMID metadata.
- First author matches.
- Year matches.
- Journal name matches.
- Volume/issue/pages are plausible.
- arXiv/bioRxiv/medRxiv ID resolves.
- Duplicate references are not accidental.
- Retractions or expressions of concern are flagged.

Classify each reference:

- Verified.
- Metadata mismatch.
- Unresolved DOI/PMID.
- Suspicious fabricated reference.
- Needs manual verification.

Never mark a reference as fabricated solely because one database fails. Check multiple sources where possible.

### Step 4: Claim-to-Citation Support Audit

Identify high-risk claims:

- Novelty claims: `first`, `novel`, `unique`, `unprecedented`.
- Superiority claims: `better than`, `outperforms`, `significantly improves`.
- Clinical claims: survival, recurrence, diagnosis, treatment response, prognosis.
- Mechanistic claims: causal pathways, immune mechanisms, tumor invasion mechanisms.
- Guideline claims: NCCN, ESMO, WHO, PRISMA, CONSORT, STROBE, ARRIVE.
- Statistical claims: significant, independent predictor, correlation, association.

For each high-risk claim:

1. Quote the claim location.
2. Identify cited reference(s).
3. Assess whether the reference directly supports the claim, indirectly supports it, or does not support it.
4. Recommend a fix: add citation, soften wording, move to discussion, or remove.

### Step 5: Figure/Table/Supplement Consistency

Check:

- Figure numbers appear in order.
- Table numbers appear in order.
- Every figure/table cited in text exists.
- Every provided figure/table is cited in text.
- Supplementary file names match text citations.
- Captions define abbreviations.
- Statistical annotations are explained.
- Scale bars, units, group sizes, and test names are present when applicable.
- Multi-panel figure labels are complete and consistent.

### Step 6: Figure, Data, and Statistical Integrity Preflight

Check visual-panel provenance, image assembly, and narrative/statistical consistency as a pre-submission quality-control task. Deterministic tabular scanning belongs to Step 7. Do not label issues as fraud. Use language such as `possible duplication`, `source-data gap`, `statistical inconsistency`, or `requires author confirmation`.

#### Figure panel provenance audit

For each figure panel where source files are available, map:

```text
Figure panel -> raw/source file -> quantification table -> statistical test -> caption/results statement
```

Flag:

- Figure panels without traceable source images or source data.
- Representative images without a matching quantification source.
- Quantification plots without clear raw values, group sizes, or test methods.
- Figure legends that do not explain scale bars, units, stains, markers, gates, or abbreviations.

#### Image reuse and assembly artifact screening

Screen for:

- Visually similar panels used for different groups, time points, experiments, or supplements.
- Possible reuse after crop, rotation, flip, brightness/contrast change, or relabeling.
- Reused Western blot loading controls across unrelated experiments without explicit explanation.
- Western blot, gel, IF/IHC, microscopy, flow cytometry, or colony images with abrupt boundaries, inconsistent backgrounds, mismatched resolution, or unclear assembly.
- Flow cytometry plots where gates, axes, compensation, event counts, or group labels are missing or inconsistent.

If pixel-level analysis is not performed, state the limitation and recommend manual review of original files or specialized tools.

#### Source data and quantification traceability

Check whether every quantitative figure has:

- Raw values or source data table.
- Clear sample size for each group.
- Definition of biological vs technical replicates.
- Description of normalization and exclusion rules.
- Consistency between plotted values, source data, captions, and results text.

Flag values that appear duplicated across supposedly independent experiments, but require author confirmation before drawing conclusions.

#### Statistical internal consistency audit

Check:

- `n` values are consistent across Methods, Results, figure legends, source data, and statistical tests.
- SD, SEM, CI, IQR, or range are clearly labeled and used consistently.
- Exact p values, significance symbols, and threshold statements match.
- ANOVA, post-hoc tests, paired/unpaired tests, nonparametric tests, and multiple-testing correction are appropriate for the stated design.
- Reported test statistics, degrees of freedom, and p values are internally plausible when available.
- Multiple comparisons, subgroup analyses, and omics-scale tests include correction methods where required.

#### Methods-results-figure consistency matrix

Build a concise matrix for high-risk experiments:

| Experiment | Methods | Results | Figure legend | Supplement/source data | Status |
|---|---|---|---|---|---|

Compare group definitions, sample sizes, treatment dose, treatment duration, assay method, normalization, statistical test, and exclusion criteria. Flag contradictions as `MAJOR` or `MANUAL` depending on whether the evidence is direct or requires author confirmation.

### Step 7: Table and Source-Data Integrity Release Gate

When quantitative tables or figure-level source-data exports are available, read and follow [references/table_source_data_gate.md](references/table_source_data_gate.md). That reference is the authoritative detector, false-positive, adjudication, reconciliation, and release protocol; do not duplicate it from memory.

Use `scripts/table_integrity_gate.py` to freeze the source-file manifest and invoke or interpret a compatible paperconan scan. The wrapper must validate the scan schema/version and reconcile the manifest against scanner-reported files and Sheets. Any missing, skipped, unparsed, incompatible, or unaccounted item is a `BLOCKER`, never a clean result.

Preserve exact evidence location and all `kept`, `demoted`, and `hidden` states. Keep review status separate from scientific impact (`CORE`, `SUPPORTING`, `PERIPHERAL`). Recompute central summaries where feasible, reconcile values across all manuscript artifacts, and require an independent benign-explanation/red-team review for unresolved high-impact signals.

Final clearance requires complete parse coverage, traceable outputs, closure of blockers and review items, and named author sign-off. Even then, state only: "No unresolved issue was detected within the assessed scope."

### Step 8: Scientific and Reporting Readiness

Depending on manuscript type, check relevant items:

#### Clinical or observational study

- IRB/ethics approval.
- Consent statement.
- Inclusion/exclusion criteria.
- Baseline characteristics.
- Missing data handling.
- Statistical methods.
- Multivariable model assumptions.
- STROBE checklist if applicable.

#### Randomized trial

- Trial registration.
- CONSORT flow diagram.
- Randomization and allocation concealment.
- Blinding.
- Sample-size calculation.
- Primary endpoint definition.

#### Systematic review/meta-analysis

- PRISMA 2020 compliance.
- Search strategy reproducibility.
- Databases and dates.
- Inclusion/exclusion criteria.
- Risk-of-bias assessment.
- Protocol registration.

#### Animal study

- ARRIVE compliance.
- Animal ethics approval.
- Randomization.
- Blinding.
- Humane endpoint.
- Sex/age/strain.
- Sample-size rationale.

#### Bioinformatics or single-cell study

- Raw data accession.
- Code availability.
- Software versions.
- Genome/reference version.
- QC thresholds.
- Normalization method.
- Batch correction/integration method.
- Clustering resolution.
- Cell annotation criteria.
- Differential expression method and thresholds.
- Multiple testing correction.
- Benchmark baselines.
- Reproducible environment.

### Step 9: Output Report

Before author clearance, apply [references/checklist.md](references/checklist.md). For a formal deliverable, fill [templates/preflight_report.md](templates/preflight_report.md); consult [examples/example-preflight-report.md](examples/example-preflight-report.md) only when an example is useful.

Produce a structured report with:

1. Executive verdict.
2. Critical blockers.
3. Major issues before submission.
4. Minor polish issues.
5. Reference audit table.
6. Claim-citation audit table.
7. Figure/table/supplement consistency table.
8. Figure/data/statistical integrity table.
9. Table and source-data release-gate coverage and findings.
10. Methods-results-figure consistency matrix.
11. Reporting-guideline checklist.
12. Manual author checklist.
13. Exact next actions.

Use severity labels:

- `BLOCKER`: do not submit before fixing.
- `MAJOR`: should fix before submission.
- `MINOR`: polish or improve if time permits.
- `MANUAL`: requires author confirmation.
- `PASS`: no issue detected.

## Helper Scripts

If available, use:

```bash
python scripts/preflight_static_scan.py <manuscript_folder_or_file> --out static_scan.tsv
python scripts/reference_audit.py <reference_file.bib> --out reference_audit.tsv
python scripts/table_integrity_gate.py --input-dir <source_data_dir> --out <table_gate_dir> --profile review
```

To interpret an existing paperconan scan without rerunning it:

```bash
python scripts/table_integrity_gate.py --scan-json <scan.json> --out <table_gate_dir>
```

The table gate requires the external `paperconan` CLI only when scanning a source-data directory. Ask before installing it into a shared environment. If it is unavailable, mark the numerical gate `NOT_ASSESSED`; do not claim a pass.

Use all script results as supporting evidence, not as final judgment.

## Final Response Style

Be direct and practical. The user is preparing for submission, so prioritize actionable fixes over abstract commentary.

Do not say "this manuscript is AI-written." Say "this phrase/file location may look like AI drafting residue and should be removed or rewritten."

Do not say "this citation is fake" unless multiple independent checks support that. Say "this reference could not be verified and requires manual confirmation."

## Minimum Output When Time Is Limited

If time or file access is limited, always provide at least:

1. Top 5 submission blockers.
2. Top 10 reference risks.
3. Top 10 AI residue/style risks.
4. Exact files or sections that need manual checking.
